# Credit Card Fraud Detection Using Machine Learning
Financial institutions process millions of transactions daily, making automated fraud detection essential. Machine learning models can help identify suspicious transactions while minimizing disruption to legitimate customers. This project builds a machine learning classifier to detect fraudulent credit card transactions using a highly imbalanced dataset. Model performance is evaulated using ROC curves, AUC, confusion metrices, and coefficient analysis.

## Business Context

Credit card fraud represents a significant financial risk for banks and payment processors. Detecting fraudulent transactions quickly is critical to preventing financial losses while minimizing disruption to legitimate customers. Fraud detection systems help financial institutions reduce losses while maintaining a positive customer experience by minimizing false alarms.

However, fraud detection is challenging because fraudulent transactions represent only a very small fraction of total activity. This creates a highly imbalanced classification problem where traditional accuracy metrics can be misleading.

Machine learning models can help identify suspicious patterns in transaction data and flag potentially fraudulent activity for further review.

## Dataset

The dataset contains anonymized credit card transactions labeled as either fraudulent or legitimate.
- Observations: a smaller subset of 1000 transactions
- Fraud cases: rare events (<1% of transactions)
- Features: anonymized PCA-transformed variables (V1–V28), transaction amount, and time

Due to GitHub file size limitations, a sample of 20,000 observations is included in this repository.

## Project Objectives

- Build a machine learning classifier to detect fraudulent transactions
- Evaluate model performance using ROC-AUC and confusion matrix metrics
- Examine model coefficients to understand influential features
- Demonstrate techniques for working with imbalanced datasets
